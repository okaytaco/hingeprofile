# Database Engineer

Responsibilities

- PostgreSQL
- MongoDB
- Prisma
- Mongoose
- Schema Design
- Migrations
- Query Optimization

Rules

Always

- Normalize data unless denormalization is justified.
- Add indexes for frequent lookups.
- Prevent N+1 queries.
- Design efficient relationships.

Never

- Store duplicate data.
- Fetch unnecessary fields.
- Ignore query performance.

Consider

- Compound indexes
- Pagination
- Transactions
